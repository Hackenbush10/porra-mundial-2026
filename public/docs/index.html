Resumen del proyecto

Aplicación web interna para que los compañeros de El País puedan rellenar sus predicciones del Mundial de fútbol 2026 (48 equipos, 12 grupos, 7 rondas) y generar un PDF personalizado con su bracket.

Cada participante introduce su nombre y sección, navega un wizard fase a fase y al final recibe un PDF descargable. Las predicciones se persisten en Supabase. Un dashboard separado (/dashboard) muestra la clasificación en tiempo real a partir de un Google Sheets puntuado manualmente.

✓
Estado actual: Desplegado y en producción en Vercel. Wizard, generación de PDF, almacenamiento en Supabase, dashboard y Apps Script de scoring operativos.

🔐 Política RLS + GRANT

04

Fases del wizard

La app guía al usuario mediante un wizard de varios pasos. Sin fechas ni sedes; solo fases.

1
Datos personales
Nombre (texto libre) y sección/departamento (texto libre).
2
Fase de grupos
Selección del 1º y 2º clasificado de cada uno de los 12 grupos (A–L).
3
Mejores terceros
Selección de 8 grupos cuyos terceros pasan a 1/16. La lógica de asignación de slots sigue la tabla oficial FIFA de 495 combinaciones C(12,8) (thirdPlaceTable.ts).
4
Ronda de 16 (1/16)
32 equipos → 16 partidos. Slots calculados automáticamente a partir de grupos + terceros.
5
Cuartos (1/8 y 1/4)
16 → 8 → 4 equipos en dos sub-fases del wizard.
6
Semifinales y Final
4 → 2 → 1 equipos. Sin partido de tercer puesto.
7
PDF y envío
Generación client-side del PDF con @react-pdf/renderer. Envío a Supabase al confirmar.
05

Lógica de terceros clasificados

La FIFA asigna los 8 mejores terceros a slots fijos del bracket de 1/16 según qué combinación de grupos avanza. Hay exactamente 495 combinaciones posibles (C(12,8) = 495).

📁 thirdPlaceTable.ts

Lookup table completa generada desde el Excel oficial de la FIFA. Mapea cada combinación de 8 grupos a los 8 slots del bracket que ocupan sus terceros.

// Ejemplo de entrada en la tabla (clave = 8 letras ordenadas) "ABCDEFGH": { vs1A: "H", vs1B: "G", vs1D: "B", vs1E: "C", vs1G: "A", vs1I: "F", vs1K: "D", vs1L: "E" }

Cada slot (vs1A…vs1L) corresponde a un partido concreto de la ronda de 16. El valor es la letra del grupo cuyo tercer clasificado ocupa ese slot.

⚠
Ver sección de bugs: los slots vs1I y vs1B estaban invertidos en la capa UI (data.ts), no en esta tabla. La tabla en sí es correcta.

07

Google Apps Script

Script instalado en el Google Spreadsheet de puntuación. Sincroniza predicciones de Supabase y calcula puntos automáticamente según los resultados reales.

📋 Hojas del spreadsheet
HojaContenidoQuién la escribe
Predicciones Todas las apuestas importadas de Supabase, con puntos calculados por fase y total Apps Script (auto)
Resultados Resultados reales del torneo, fase a fase Manual (tú)
⚙️ Menú personalizado y triggers
  • Sincronizar predicciones — trae todos los registros de apuestas desde Supabase
  • Recalcular puntos — compara predicciones vs. Resultados y actualiza columnas de puntos
  • Actualizar todo — sincroniza + recalcula en un paso
  • Activar/desactivar auto-refresh horario — trigger basado en tiempo
⚠
El script requiere la service_role JWT key de Supabase (no la anon/publishable key). Se configura directamente en el editor de Apps Script, no se expone al cliente.
🔑 Acceso a Supabase desde Apps Script
// Configuración en Apps Script const SUPABASE_URL = 'https://[proyecto].supabase.co'; const SERVICE_KEY = 'eyJ...'; // service_role JWT // Fetch a la tabla apuestas UrlFetchApp.fetch(SUPABASE_URL + '/rest/v1/apuestas', { headers: { 'apikey': SERVICE_KEY, 'Authorization': 'Bearer ' + SERVICE_KEY } });

La service_role key se obtiene en Supabase → Settings → API → service_role (secret). Nunca en las variables de entorno de Vercel ni en el repositorio.

09

Bugs resueltos

🐛 Bug 1 — Slots vs1I / vs1B invertidos en data.ts
CampoDetalle
SíntomaLos terceros clasificados de las combinaciones que usaban slots vs1B e vs1I aparecían en el bracket equivocado
CausaLos slots estaban invertidos en las definiciones del bracket en data.ts (capa UI), no en thirdPlaceTable.ts (que es correcta)
FixIntercambio de los valores de vs1I y vs1B en data.ts
VerificaciónCombinación ABCDEFGH: el tercero de A va al slot correcto según la tabla FIFA
🐛 Bug 2 — Clave anon usada en Apps Script en lugar de service_role
CampoDetalle
SíntomaEl Apps Script no encontraba ninguna predicción en Supabase (0 resultados)
CausaSe estaba usando la sb_publishable_... (anon key) en lugar del JWT service_role. La anon key solo tiene permiso de INSERT, no de SELECT (según la política RLS)
FixReemplazar la clave en el editor de Apps Script por la service_role JWT (Supabase → Settings → API)

Operaciones y mantenimiento

🚀 Flujo de despliegue
  1. Push a main en GitHub
  2. Vercel detecta el push y lanza build automático
  3. Variables de entorno (NEXT_PUBLIC_SUPABASE_URL, NEXT_PUBLIC_SUPABASE_ANON_KEY) ya configuradas en el panel de Vercel
  4. Deploy en producción sin intervención manual
📅 Cambio de política Supabase — Oct 2026

A partir de octubre 2026, Supabase requiere grants explícitos para que el rol anon acceda a tablas del schema public vía Data API, además de las políticas RLS.

✓
El GRANT INSERT ON public.apuestas TO anon ya fue ejecutado en mayo 2026. No se requiere acción adicional.
🔄 Actualizar clasificación durante el torneo
  1. Abrir el Google Spreadsheet
  2. Rellenar los resultados reales en la hoja Resultados
  3. Menú Porra → Actualizar todo (o esperar el trigger horario si está activo)
  4. El dashboard en /dashboard refleja los cambios al pulsar el botón de actualizar
⚡ Notas técnicas relevantes
  • La anon key (prefijo sb_publishable_) no funciona para lecturas server-side en Supabase; solo para INSERTs desde el cliente
  • El CSV de Google Sheets siempre debe proxearse por app/api/scores/route.ts con cache: 'no-store'
  • El PDF se genera completamente en el cliente — sin carga en el servidor
  • No hay autenticación de usuarios — la app es completamente pública y anónima
  • El spec doc (PLAN_PORRA_MUNDIAL_2026.md) se usó como contexto persistente en todos los prompts de Claude Code durante el desarrollo
